# 04 - C++ 系统底层

> 这一层负责和 Windows 系统"打交道"，让宠物窗口能置顶、透明、穿透，还能监听系统事件。目前只支持 Windows，其他平台需要另外实现。

---

## 4.1 窗口管理模块

### 4.1.1 实现获取 Unity 窗口句柄
- [ ] 学习 `FindWindow` / `EnumWindows` API
- [ ] 编写通过进程名查找窗口句柄的函数
- [ ] 输入：进程名（如 `AIDesktopPet.exe`）
- [ ] 输出：`HWND` 窗口句柄
- [ ] 验证：能正确找到 Unity 窗口

### 4.1.2 实现窗口置顶 / 取消置顶
- [ ] 学习 `SetWindowPos` 与 `HWND_TOPMOST` / `HWND_NOTOPMOST`
- [ ] 封装 `SetWindowTopmost(hwnd, topmost)` 函数
- [ ] 验证：调用后窗口置顶状态正确切换

### 4.1.3 实现窗口透明度设置
- [ ] 学习 `SetLayeredWindowAttributes` API
- [ ] 封装 `SetWindowTransparent(hwnd, alpha)` 函数（0-255）
- [ ] 验证：透明度可平滑调节

### 4.1.4 实现点击穿透控制
- [ ] 学习 `SetWindowLong` 修改窗口扩展样式
- [ ] 添加 / 移除 `WS_EX_TRANSPARENT` 和 `WS_EX_LAYERED`
- [ ] 封装 `SetWindowClickThrough(hwnd, enable)` 函数
- [ ] 验证：开启穿透后点击透明区穿透，关闭后正常点击

### 4.1.5 实现异形窗口设置
- [ ] 学习 `SetWindowRgn` API
- [ ] 根据传入的点数组创建 `HRGN` 区域
- [ ] 封装 `SetWindowRegion(hwnd, points[], count)` 函数
- [ ] 验证：窗口形状可自定义（如圆形、宠物轮廓）

### 4.1.6 实现窗口位置 / 大小控制
- [ ] 封装 `GetWindowPos(hwnd)` 获取位置和尺寸
- [ ] 封装 `SetWindowPos(hwnd, x, y, w, h)` 设置位置和大小
- [ ] 验证：Unity 能正确调用并控制窗口

## 4.3 P/Invoke 封装（供 C# 调用）

### 4.3.1 设计 C# 调用接口
- [ ] 在 C# 中编写 `DllImport` 声明
- [ ] 统一调用约定（`CallingConvention.Cdecl`）
- [ ] 参数类型映射（`HWND` → `IntPtr`，`BOOL` → `bool` 等）
- [ ] 验证：C# 能正确加载 DLL

### 4.3.2 实现 DLL 导出函数
- [ ] 在 C++ 中使用 `extern "C" __declspec(dllexport)` 导出函数
- [ ] 导出的函数列表：
  - `GetUnityWindowHandle`
  - `SetWindowTopmost`
  - `SetWindowTransparent`
  - `SetWindowClickThrough`
  - `SetWindowRegion`
  - `GetWindowPos`
  - `SetWindowPos`
  - `AddTrayIcon`
  - `RemoveTrayIcon`
  - `RegisterGlobalHotkey`
  - `SetStartup`
- [ ] 验证：使用 Dependency Walker 查看导出函数

### 4.3.3 集成到 Unity 项目
- [ ] 将编译好的 `native_plugin.dll` 放入 `Plugins/x86_64/`
- [ ] 在 Unity 中创建 `NativePlugin.cs` 封装层
- [ ] 在启动时自动调用窗口初始化
- [ ] 验证：Unity 运行后窗口自动变为无边框透明
